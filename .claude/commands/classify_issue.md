# Github Issue Command Selection

Based on the `Github Issue` below, follow the `Instructions` to select the appropriate command to execute based on the `Command Mapping`.

## Instructions

- Read the `Github Issue` and decide which single command best fits it, using the `Command Mapping`.
- **Output ONLY the command token on a single line** (e.g. `/feature`) — nothing else.
- Your entire response must be exactly one of: `/chore`, `/bug`, `/feature`, or `0`.
- Do NOT use any tools. Do NOT read, search, or write files. Do NOT research the codebase.
- Do NOT explain your reasoning, add markdown, or write a summary. Respond with the token only.

## Command Mapping

- Respond with `/chore` if the issue is a chore.
- Respond with `/bug` if the issue is a bug.
- Respond with `/feature` if the issue is a feature.
- Respond with `0` if the issue isn't any of the above.
